# Contribution Guidelines

We welcome contributions to the Agendaspa project! Below are some guidelines to help you get started.

## How to Contribute
1. **Fork the repository**: This is the first step to contributing to our project. You can create your own copy of the repository to work on.
2. **Create a new branch**: It's best to create a new branch for each feature or fix you are working on. This helps keep our main branch clean. The naming convention for branches can be `feature/your-feature-name` or `bugfix/your-bugfix-name`.
3. **Make your changes**: Work on your contribution in your new branch.
4. **Commit your changes**: Make sure to commit only related changes. Use clear commit messages that express the intent of your contribution.
5. **Push your changes**: After you have committed your changes, push them up to your forked repository.
6. **Open a pull request**: Once you have pushed your code, you can open a pull request against the main repository.

## Pull Request Guidelines
- Clearly describe the issue your pull request aims to resolve, or the feature you are adding.
- If you're adding a new feature, include appropriate tests.
- Ensure your code adheres to our coding standards.
- Be ready to respond to feedback and make adjustments if needed.

## Code of Conduct
Please make sure to read and adhere to our [Code of Conduct](link-to-code-of-conduct) to foster a welcoming environment for all contributors.

We appreciate your contributions and look forward to collaborating with you!
